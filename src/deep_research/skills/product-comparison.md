# 产品对比方法论

## 信息源优先级
- 一手：厂商官网（spec sheet / 发布会 / 技术文档）、官方支持库 / changelog
- 二手：专业评测媒体（The Verge / DXOMARK / Tom's Hardware / RTINGS / Wirecutter / AnandTech）、用户社群（reddit / 知乎 / 行业论坛）
- 价格与库存：主流零售商（Amazon / 京东 / Apple Store / 厂商直营店）
- 排除：SEO 内容农场聚合、纯广告软文、超过一年的旧评测（除非作为时间基线）

## 查询模式
**第一次搜索必须使用以下模板之一**（把占位符替换为本次具体产品），不要降级为通用关键词。只有当该模板召回不足时，再放宽：
- `"<产品 A>" vs "<产品 B>" review`（如 `iPhone 17 Pro vs Pixel 10 review`）
- `"<产品>" spec sheet site:<官方域名>`
- `"<产品>" "<关键维度>" benchmark 2026`
- `"<产品>" review reddit OR forum`（找一线用户痛点）

## 停止判据
- 每个候选在核心维度（性能 / 价格 / 兼容 / 售后）均有 ≥2 个独立来源支撑
- 关键论点有同向交叉证据（厂商 spec × 第三方实测同向）
- 最近两次搜索结果已落在已知信息集合内

## 引用规范
- 评测：媒体名(日期). 标题. URL
- spec 必带型号 / 版本号 + 测试条件（固件、配置、环境）
- 价格附币种 + 抓取日期
